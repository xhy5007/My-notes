# **一、重发布（路由引入）**

## **1、背景**

一个网络拓扑中存在多种不同的路由协议，为了使多种不同的路由协议间能相互通信，出现了路由引入



![image](https://share.note.youdao.com/yws/public/resource/a54c121161e06e2d61673c9a8773f722/xmlnote/WEBac027e9ad743b078392b695a24f434b8/WEBRESOURCEe4a4a7ace76368e3ee60d2d793559111/29815)



## **2、路由引入的情形**

在边界设备（ASBR设备）上，同时运行多种路由协议。

需要将一种路由协议引入到另一种路由协议中学习；把路由在同种协议的不同进程间的引入

![image](https://share.note.youdao.com/yws/public/resource/a54c121161e06e2d61673c9a8773f722/xmlnote/WEBac027e9ad743b078392b695a24f434b8/WEBRESOURCE4179388ac4edd969e4d44dc22f804fd5/29816)

注意：关注两种协议起始度量值，引入时原路由



## **3、规则**

（1）将A协议发布到B协议中，在ASBR中的B协议上配置

（2）将A协议发布到B协议中，所有通过A协议学习到的路由及A的直连路由，B都可以学习到



重发布举例：R1-R3均通过x.x.x.x/32建邻

![image](https://share.note.youdao.com/yws/public/resource/a54c121161e06e2d61673c9a8773f722/xmlnote/WEBac027e9ad743b078392b695a24f434b8/WEBRESOURCE7374658a0ab18c9028f1e0717f105c2d/35593)



举例:1：将OSPF路由引入RIP，在rip进程视图下：[R2-rip-1]import-route ospf 1，

如果不指定cost，路由重发布进RIP后，cost缺省为0 ；包括环回口。每经过一跳，cost加1，cost最大是15



举例:2：将RIP路由引入OSPF：在OSPF进程视图下：[R2-ospf-1]import-route rip 1

如果不指定cost，路由重发布进ospf后，cost缺省为1；type类型为E2；包括环回口 ；每经过一个路由器，cost不会改变(不计算ospf域内链路的cost)。

[R3-ospf-1]import-route rip 1 type 1   -----计算ospf域内链路的cost

![image](https://share.note.youdao.com/yws/public/resource/a54c121161e06e2d61673c9a8773f722/xmlnote/WEBac027e9ad743b078392b695a24f434b8/WEBRESOURCE202c7e21c6fc00d37b25bc111b7a5f11/35601)



举例3：将静态引入到动态协议中：

[r2-rip-1]import-route static

[r2-ospf-1]import-route static



举例4：将直连路由导入到动态路由协议中：

[r2-rip-1]import-route direct

[r2-ospf-1]import-route direct



## **4、修改起始度量值命令**

进程当中对全局进行修改：

[r2-rip-1]default-cost 2      //对引入的路由修改起始度量值

[R1-ospf-1]default cost 5

针对本次重发布进行修改：

[r2-rip-1]import-route ospf 1 cost 3

[R1-ospf-1]import-route rip cost 10



## **5、多协议网络规划**

一般情况下，建议跑单协议

路由协议的规划规则：

### **（1）从边缘引入核心；**



![image](https://share.note.youdao.com/yws/public/resource/a54c121161e06e2d61673c9a8773f722/xmlnote/WEBac027e9ad743b078392b695a24f434b8/WEBRESOURCE76b6deb22bbf4c0e9bfca8b519ab6c56/29818)



因为边缘设备的性能低，承受能力差，核心网络的路由器性能比较好，承受能力强。需要将边缘路由引入核心，核心路由不需要引入边缘，这时建议在边界路由器上下发一条缺省路由，以保证边缘的路由器可以通过这条缺省路由访问核心网络。

### **（2）从IGP引入到BGP**

![image](https://share.note.youdao.com/yws/public/resource/a54c121161e06e2d61673c9a8773f722/xmlnote/WEBac027e9ad743b078392b695a24f434b8/WEBRESOURCE2a0de7c5f30e4526ab4f7f83974a988a/29819)



原因：

公网上路由太多了，要是将BGP公网上的路由引入内网的ospf中，会造成内网路由器承受不住。

所以一般建议从IGP引入到BGP，但我们仍然有学习BGP上路由的需求，所以从IGP引入BGP时，建议IGP内部通过配置默认路由或者路由聚合达到进入其他AS的路由。

环路解决方法：使用Tag来进行选择性路由引入

在路由引入时加上Tag标记值，根据tag标记值来选择引入的路由

配置示例：将OSPF路由引入RIP中时，打上标记值

```ada
抓取流量：
[R2]acl 2000
[R2-acl-basic-2000]rule permit source 100.1.1.0 0.0.0.255
 
用路由策略修改路由属性：
[R2]route-policy aa permit node 10
[R2-route-policy]if-match acl 2000
[R2-route-policy]apply tag 99
 
[R2]route-policy aa permit node 20
 
调用策略
[R2-rip-1]import-route ospf route-policy aa
 
在R3做个筛选，对于含有99标记的路由不引入
route-policy bb deny node 10
if-match tag 99
route-policy bb permit node 20
[R3-ospf-1]import-route rip route-policy bb
```



次优路径解决方法：合理规划引入路由的初始度量值，避免引入次优路由

规划引入路由的优先级，避免引入次优路由



```ada
抓取流量：
[R3]acl 2000
[R3-acl-basic-2000]rule permit source 100.1.1.0 0.0.0.255
写路由策略：
[R3]route-policy qq permit node 10
[R3-route-policy]if-match acl 2000
[R3-route-policy]apply preference 151
[R3]route-policy qq permit node 20
调用路由策略：
[R3-rip-1]preference route-policy  qq
```

## **5、路由策略的匹配规则**

### **（1）规则**

1. 路由信息到达时，检查是否配置了路由策略，如果配置，则进入节点匹配；否则放行；
1. 检查路由策略第一个节点的if-match条件，匹配则检查节点动作是permit还是deny，如果是permit则进一步检查apply子句，有apply子句则执行路由属性修改并放行。没有apply子句则不修改属性并放行；如果是deny则不能通过。如果没有匹配该节点的if-match条件，则检查下一个节点
1. 最后一个节点仍不匹配，则拒绝该路由信息通过。



### **（2）基本配置**

#### **a.创建一个****route-policy****节点**

[R1]route-policy route-policy-name { permit | deny } node node编号

**节点之间的关系是“或”的关系**，如果通过了其中一个节点，就意味着通过该路由策略，不再对其他节点进行匹配

#### **b.配置****if-match语句**

使用if-match命令可定义匹配条件，所匹配的对象是路由信息的一些属性，例如路由的目的网络地址或掩码长度、度量值、标记或下一跳IP地址等

#### **c.****配置apply 语句**

执行语句，在 Route-Policy的节点视图下，使用apply命令指定需执行的动作，这些动作主要是对所匹配的路由的某些属性进行修改

apply执行动作表

![image](https://share.note.youdao.com/yws/public/resource/a54c121161e06e2d61673c9a8773f722/xmlnote/WEBac027e9ad743b078392b695a24f434b8/WEBRESOURCE3a891681e1bcdbb942123bccd346dba3/17610)



1. 设置路由的度量值:**apply cost  cost**
1. 设置OSPF的度量值类型:**apply cost-type { type-1 / type-2 }**
1. 设置路由的标记:**apply tag tag**

注：一个节点中可以不包含任何 apply语句，此时该节点不用于设置路由的属性



##### **多个apply语句的执行规则**

**（1）叠加关系**（大多数情况）：

    1. 多个apply语句会**同时生效**，共同修改路由属性
    1. 每个apply修改不同的属性，不会相互覆盖

```abap
 
抓流量：
[R4]acl  2000 
[R4-acl-basic-2000]rule permit source 50.50.50.0 0.0.0.255
 
做策略：
[R4]route-policy POLICY1 permit node 10
[R4-route-policy]if-match acl 2000
[R4-route-policy]apply cost 50   # 修改cost值
[R4-route-policy]apply tag 100   # 添加tag标记
 
[R4]route-policy POLICY1 permit node 20     //配置空节点放行其他网段
 
调用策略：
[R4-ospf-1]import-route rip route-policy POLICY1  
```

![image](https://share.note.youdao.com/yws/public/resource/a54c121161e06e2d61673c9a8773f722/xmlnote/WEBac027e9ad743b078392b695a24f434b8/WEBRESOURCE68052b78f1b29c5c74baf47c24ebb6ef/43231)



**（2）覆盖关系**（相同属性时）：

    1. 对**同一个属性**多次apply时，**最后一个**生效
    1. 前面的同属性apply会被覆盖

```abap
 [R4]route-policy POLICY1 permit node 10
 [R4-route-policy]if-match acl 2000
 [R4-route-policy]apply tag 100   # 添加tag标记
 [R4-route-policy]apply cost 50     # 这个会被覆盖
 [R4-route-policy]apply cost 100     # 只有这个cost值会生效
```

1. route-policy仅用于属性修改，需要配置空节点；
